# Hands-On: AI Agent Work in VS Code

VS Code's built-in AI assistant (GitHub Copilot Chat) has an **Agent mode** that does exactly this — it reasons about a task, decides on an action (like running a command or editing a file), observes the result, and repeats until it's done. Every one of those steps will appear directly in your chat window.


---

## What You'll Need

- VS Code installed
- The GitHub Copilot extension installed and signed in
- A project folder open in VS Code (any empty folder will do — we'll create one below)

If you're not sure whether Copilot is set up, check the bottom status bar of VS Code. If you see a Copilot icon and your GitHub username, you're ready to go.

---

## Step 1 — Create a Practice Folder

1. On your computer, create a new empty folder called `agent-practice`
2. In VS Code, go to **File → Open Folder** and open `agent-practice`
3. Open a terminal inside VS Code (**Terminal → New Terminal**) and run:

```bash
git init
```

> **Why?** Agents work best when your project is under version control. This way, if the agent does something you don't like, you can easily undo it.

---

## Step 2 — Open Copilot Chat and Switch to Agent Mode

1. Open the Chat view: press **Ctrl+Alt+I** (Windows/Linux) or **Cmd+Ctrl+I** (Mac)
2. At the top of the chat panel, find the **mode dropdown**. It usually shows "Ask" by default
3. Switch it to **Agent**

You should now see three modes available: **Ask** (just answers questions, never touches files), **Edit** (changes one file at a time), and **Agent** (can read, write, run commands, and loop on its own — this is the one we want).

> **Connecting back to the lesson:** "Ask" mode is like a normal chatbot — it only generates text. "Agent" mode is what we've been discussing all lesson: the AI can take **actions** (running commands, editing files) and **observe** the results, then decide what to do next.

---

## Step 3 — Give the Agent a Small, Multi-Step Task

This is the important part. We want a task that **requires more than one step**, so you can actually watch the Thought → Action → Observation loop happen more than once.

Type this prompt into the chat:

```
Create a file called numbers.txt containing the numbers 1 to 20, one per line.
Then write a js script called sum_numbers.js that reads numbers.txt and
prints the sum of all the numbers. Run the script and show me the result.
```

Press Enter and watch what happens.

---

## Step 4 — Watch the Loop Happen

As the agent works, pay close attention to the chat. You should see it do roughly the following, **in this order**:

| What you'll see in the chat | What it maps to from the lesson |
|---|---|
| The agent explains what it's about to do ("I'll start by creating numbers.txt...") | **Thought** — the AI reasoning about what to do next |
| The agent proposes creating/editing `numbers.txt` | **Action** — a structured request to use a "file editing" tool |
| You may be asked to confirm running a terminal command | **Action** — a structured request to use a "run terminal command" tool |
| The agent shows the terminal output (or file contents) | **Observation** — the real result, produced by your actual computer, fed back to the AI |
| The agent says something like "Now I'll create sum_numbers.js..." | **Thought** again — reasoning continues with the new information |
| ...and so on until the final result is shown | **Final answer** |

> **Key thing to notice:** The AI itself never actually wrote to your disk or ran node. It generated a *request* to do those things. VS Code's Copilot extension is the "code" we talked about in the lesson — it's the part that actually executes the action and reports back the real result.

---

## Step 5 — Try Breaking It (On Purpose)

Now let's see the agent **recover from a problem** — this is where the loop really shines.

Type this prompt:

```
Modify sum_numbers.js so that it also prints the average of the numbers,
rounded to 2 decimal places. Then run it again.
```

If there's a bug (a typo, wrong file path, etc.), watch what the agent does:

- Does it notice the error message from the terminal?
- Does it reason about *why* the error happened?
- Does it fix the code and try again automatically?

This back-and-forth — try something, see it fail, reason about the failure, try again — **is the ReAct loop working exactly as described in the lesson**, just happening live in front of you.

## Step 6 — Reflection Questions


1. In Step 4, which parts were generated by the AI (text), and which parts were performed by your actual computer (real actions)?
2. Was the agent's first attempt always correct? If not, how did it recover?
3. The agent had access to tools like "edit a file" and "run a terminal command." How is this similar to the `get_weather` and `search_flights` tools from `demo_agent.html`?

## Optional Challenge — A Slightly Bigger Task

If you have extra time, try giving the agent this prompt and see how it plans and executes a multi-file task:

```
Create a simple command-line quiz game in js. It should have a file
quiz_questions.js with a list of at least 5 trivia questions (with answers),
and a file quiz.js that imports the questions, asks them one at a time,
keeps score, and prints the final score at the end. Run it so I can see
it working, then describe what it printed.
```
